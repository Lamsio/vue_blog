# 依赖管理

![[Pasted image 20220617164506.png]] 上述配置是在父工程pom.xml中的，用于对子工程可选依赖进行管理。 意思是某些子工程可能需要用到junit库，当他们在自己的pom.xml配置junit依赖时，会根据父工程配置的依赖管理进行处理，也就是说，即使子工程打算依赖junit库时，不需要配置版本，只需要配置其artifactID和groupID即可，这样就会按照父工程中的依赖管理进行处理了。
